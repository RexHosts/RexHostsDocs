Discord.JS Bots Last updated: April 4th, 2022

Discord.JS is a powerful Node.js module that allows you to interact with the Discord API very easily. It takes a much more object-oriented approach than most other JS Discord libraries, making your bot's code significantly tidier and easier to comprehend. See https://discord.js.org.

RexHosts offers generic discord.js bot hosting. By generic, we mean you can host almost any bot that is built discord.js. Whether you want to pull a bot from a GitHub repository or upload your own bot, we can help make it happen. This documentation will go through 2 different methods of hosting a discord.js bot.

Please be aware that this documentation doesn't provide support for discord.js programming. This documentation is going to assume you have a bot already!

Prerequisites

  • A bot picked out (or developed).
    • You can find lots of open source discord.js bots on GitHub. Click here to see them!
    • Alternatively, you can have your bot on a local machine ready to be uploaded.
  • (Optional) Discord.JS hosting from RexHosts. The steps in this documentation will walk you through the setup steps when ordering.

Method 1: GitHub Repository Bot Last updated: April 4th, 2022

This method is the preferred way of getting your bot deployed on your ordered server. You can set it so that everytime the bot comes online, it will pull the latests updates from the GitHub repository.

Ordering Your Service

As mentioned above, ensure that you have your bot picked out on GitHub prior to ordering!

Step 1

Head over to https://rexhosts.com and sign in to your account. If you do not have an account, you check this doc on how to create an account.

Step 2

Click the "Bots" tab in the navigation panel, it will show a dropdown - select Discord.JS Bots. It will bring you to the order page for the discord.js service. Click order on the package you wish to order!

Step 3

You will be brought to the service configuration page which we will walk you through next.

Configuring Your Service

This section will cover on which field means when you arrive at the order configuration page.

Fields

There are a total of 7 fields on the configuration page. They look like this...

  • Upload or GitHub
    • This field is asking if you want to upload your bot files to the server when it is installed, or if you want to use a GitHub repository. In this section, we're going to use a GitHub repository.
  • GitHub Address
    • This field is asking you for the GitHub repository address. The GitHub repo address can be found when you're inside a repositories main page (where main file structure is shown).
      • Copy the address and head back to the order configuration page and enter it into the GitHub Address field.
  • GitHub Username
    • This field is asking you for your GitHub username. This is required if your the owner of the repository AND the repository is set to private.
        • This isn't required if it's a public repository, if thats the case, you can leave this blank!
  • GitHub Access Token
    • This field is asking you for an access token. An access token acts as a password in which you can use for services such as Discord.JS to access your private repositories without having to use your accounts password. You can find out more about access tokens here. You can skip this step if your repository is accessible by the public.
      • To create an access token, you must do the following (information grabbed from GitHub docs)
        1. Verify your email address, if it hasn't been verified yet.
        2. In the upper-right corner of any page, click your profile photo, then click Settings.
        3. In the left sidebar, click Developer settings.
        4. In the left sidebar, click Personal access tokens.
        5. Click Generate new token.
        6. Give your token a descriptive name.
        7. To give your token an expiration, select the Expiration drop-down menu, then click a default or use the calendar picker. This is optional!
        8. Select the scopes, or permissions, you'd like to grant this token. To use your token to access repositories from the command line, select repo. In this case, all we need to do is select the "repo" box and thats it.
        9. Click Generate token.
        10. You can then take that token, go back to the configuration page on RexHosts and enter the personal access token that was generated, into the GitHub Access Token field!
      Warning: Treat your tokens like passwords and keep them secret. When working with the API, use tokens as environment variables instead of hardcoding them into your programs.
  • Install Branch
    • This field is asking you for the branch name you want to pull from. If left blank (recommended), it will pull from the main branch. If you know the identifier to the branch you would like to use, simply copy the branch name and paste it into the Install Branch field.
  • Auto Update
    • This field asks if you want to auto update your bot on every start. This only works if your using this GitHub repository method. Everytime you restart your bot through the console, it will check for changes in the GitHub repo and apply updates if there are any.
  • Start Bot File
    • This field asks for the file name of the bot start. This is is a must in order for the bot to start. Usually the bot start files are labelled index.js or bot.js, but could be named something different. Check the repository for the main start file or in the bots respective documentation for the start file, normally it will tell you how to start the bot manually using a command like: $ node index.js - index.js being the start file.
      • NOTE: This field is case sensitive and should match your bot start file name exactly. Any mistakes may result in the bot not starting!
That's it for the configuration part of the bot setup. You can make changes to things such as file names, configuration files, etc in the file manager of the RexHosts Console. You can also change the config variables that were mentioned above, such as disabling auto update, repo name, repo keys, etc.
